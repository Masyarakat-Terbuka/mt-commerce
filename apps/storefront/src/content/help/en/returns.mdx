Return policies — what can be returned, within how many days, and who covers shipping back — are set by each store. There is no single return policy that applies to every mt-commerce store.

This page describes the return flow the platform supports, so buyers know what happens after a return is opened.

## Checking the store's policy

A store's return policy is usually linked from the footer or the store's help section. Read it before purchasing to avoid misunderstandings.

Things commonly defined in a store policy:

- Which items are eligible (e.g. custom or hygienic items often excluded)
- The return window after delivery
- Item condition required for return (seal, packaging, label)
- Who pays return shipping
- Refund method and timing

## Return flow on mt-commerce

Once the store operator approves a return request, the order moves through the return states:

1. **Requested** — buyer submits the request with reason and photos if relevant
2. **Approved** — operator approves and sends the return address
3. **In transit** — the item is shipped back with tracking
4. **Received** — the operator confirms the item arrived in acceptable condition
5. **Resolved** — refund or exchange is processed

Each transition triggers an automatic email notification. The full history is visible on the order page.

## Refunds

Refunds go back to the same payment method used for the purchase. For manual bank transfer, funds are sent to the account the buyer specifies. For gateway payments, funds are returned through the same gateway.

The time involved depends on the method and the banks involved — typically 1–7 business days.

## When a request is declined

Operators can decline a request with a reason — for example, if the return window has passed or the condition does not match the policy. Buyers can discuss directly with the store through its support channel. mt-commerce as a platform does not mediate disputes between buyers and stores.
